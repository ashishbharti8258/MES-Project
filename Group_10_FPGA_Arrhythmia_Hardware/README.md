# ECG Hardware FPGA Design & Deployment

Welcome to the Hardware Module of the ECG FPGA Classification Project. This directory contains the entire Xilinx-based hardware pipeline required to accelerate the trained ECG CNN on an FPGA, specifically targeting a Zynq-7000 SoC (like the PYNQ-Z2). 

This module heavily relies on the outputs generated by the [Software Module](../software_model_training/README.md). You must have the quantized C++ weight headers ready before beginning synthesis.

## Directory Overview

* **`cpp_quantized_weights/`**: The bridge between software and hardware. This folder holds the `weights.h` and associated C++ headers generated by the Python notebook. These files contain the hardcoded, INT8 quantized arrays that the Vitis HLS IP will use to compute the convolutions.
* **`vitis_hls_ip_core/`**: The Vitis High-Level Synthesis (HLS) project. This folder contains the C++ source code for the CNN architecture. Vitis HLS will consume the C++ code and the quantized weights to generate a custom RTL (Register Transfer Level) IP block optimized for the FPGA fabric.
* **`vivado_system_project/`**: The Vivado block design project. Here, the custom CNN IP core generated by Vitis HLS is integrated with the Zynq Processing System (PS). Direct Memory Access (AXI DMA) blocks are used to stream ECG data from the ARM processor to the CNN accelerator and read back the predictions. This project ultimately generates the bitstream (`.bit`) and hardware handoff (`.hwh`) files.
* **`pynq_deployment_files/`**: The final deployment zone. This contains the Jupyter Notebook (`pynq_hardware_inference.ipynb`) designed to run on the physical PYNQ board's ARM processor. It handles loading the custom bitstream, managing memory buffers, streaming data to the FPGA fabric, and displaying the real-time classification results.
* **`MES_Project_Report.pdf` & `Project_Presentation.pdf`**: The official documentation and presentation slides detailing the hardware synthesis and deployment results.

## Hardware Workflow

### Phase 1: High-Level Synthesis (Vitis HLS)
1. Ensure the `cpp_quantized_weights/` directory is populated with the headers from the software training phase.
2. Open the `vitis_hls_ip_core/` project in Vitis HLS.
3. Run C Simulation, C Synthesis, and Export RTL. This will package your CNN as an IP core.

### Phase 2: System Integration (Vivado)
1. Open the `vivado_system_project/` in Xilinx Vivado.
2. Add your newly exported Vitis HLS IP to the IP Catalog.
3. Open the block design to verify the AXI stream connections between the Zynq PS, the AXI DMA, and your CNN accelerator.
4. Run Synthesis, Implementation, and **Generate Bitstream**.
5. Export the hardware (`.xsa` or `.hwh` depending on your PYNQ version) along with the `.bit` file.

### Phase 3: Physical Deployment (PYNQ)
1. Power on your PYNQ board and connect it to your local network.
2. Upload the generated `ecg_cnn.bit`, the `.hwh` file, and the `pynq_hardware_inference.ipynb` notebook to the PYNQ Jupyter environment.
3. Run the notebook to initialize the overlay and watch the FPGA classify ECG arrhythmias in milliseconds!
